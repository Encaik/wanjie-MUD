# 炼制系统

## 概述

炼制页面（`CraftPage`）合并旧炼丹、炼器两个独立页面，并正式化炼制逻辑。利用统一物品系统的模板和实例管理能力，替代旧占位实现。

## 功能需求

### FR-1: 炼丹

- 从 `modules/item/` 的消耗品模板中筛选可炼制的丹药（`ItemTemplate.category === 'consumable'` 且可炼制标记）
- 显示配方列表，每项展示：
  - 丹药名称、稀有度、效果描述
  - 所需材料清单（模板ID × 数量）
- 检查背包材料是否足够（`hasEnough`）
- 点击炼制 → 消耗材料 → 调用 `generateItemInstance(templateId)` 生成新物品实例 → 加入背包

### FR-2: 炼器

- 从 `modules/item/` 的装备模板中筛选可锻造的装备
- 显示配方列表，每项展示：
  - 装备名称、稀有度、属性预览
  - 所需材料清单
- 点击锻造 → 消耗材料 → 调用 `generateItemInstance(templateId)` → 加入背包

### FR-3: 配方管理

- 配方通过检查模板的 `ext` 字段来判定：
  - 消耗品模板的 `ext` 中定义炼制配方
  - 装备模板的 `ext` 中定义锻造配方
- 配方是否需要额外数据字段在设计时确定

## 路由

- 路径: `/game/craft`
- 导航位置: "更多" → 炼造组 → 炼制
