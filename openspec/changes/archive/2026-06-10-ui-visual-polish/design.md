## Context

当前项目 UI 结构清晰但视觉效果存在不一致问题。项目使用 Tailwind CSS 4 + shadcn/ui + React 19，组件位于 `src/components/game/`（76 个 `.tsx` 文件）、`src/components/pages/`（5 个页面组件）和 `src/components/layout/`。现有组件使用 Tailwind 内联类名，shadcn/ui 提供基础 UI 组件。

视觉问题可能包括：间距不统一、颜色对比度不足、暗色主题一致性差、中文排版未优化、空状态缺失、响应式布局不够流畅、交互反馈（hover/active/focus）不完整。

## Goals / Non-Goals

**Goals:**
- 逐页审查所有可见页面和面板，记录视觉问题
- 建立视觉问题优先级分类体系（高/中/低）
- 修复高优先级和中优先级视觉问题
- 建立 UI 视觉检查清单，供后续开发参考
- 所有修改仅限于 Tailwind CSS 类名、JSX 布局结构调整，不动功能逻辑

**Non-Goals:**
- 不修改任何功能逻辑（hooks、lib、storage）
- 不新增或删除组件
- 不修改 shadcn/ui 基础组件
- 不进行性能优化（除非视觉修改自然带来）
- 不添加新的依赖包

## Decisions

### D1: 审查方式 — 代码审查为主，运行验证为辅
- **决策**：通过阅读 `.tsx` 源码审查视觉问题，辅以 `pnpm dev` 实际运行截图对比
- **原因**：当前环境无浏览器自动化工具，纯代码审查可以发现 80% 的视觉问题（间距、颜色、排版、状态覆盖）
- **替代方案**：使用 Playwright 截图对比（但需要额外配置，超出本次范围）

### D2: 优先级分类标准
- **高优先级**：影响可读性（对比度不足）、布局错位（元素重叠）、功能可见性受损（按钮/链接不明显）
- **中优先级**：视觉不一致（同类元素间距不同）、缺失反馈状态（hover/pressed/focus）、空状态无提示
- **低优先级**：动效缺失、微调优化、"锦上添花"类改进
- **原因**：确保有限时间内修复最关键的问题

### D3: 问题记录格式
- 采用统一格式：`[文件:行号] 问题描述 → 建议修复方案`
- 按页面/面板分组，便于批量修复和复查
- **原因**：方便追踪和验证

### D4: 修复策略 — 逐文件、小步提交
- 每个文件修复后运行 `pnpm lint` 和 `pnpm ts-check` 确认无破坏
- 按面板分组提交，每组修复后独立验证
- **原因**：最小化风险，方便回滚

### D5: 视觉标准基线
- 间距系统：统一使用 Tailwind 间距（`gap-2/4/6` 等），禁止混用自定义 px 值
- 颜色系统：统一使用 Tailwind 语义色（`text-foreground`、`bg-card` 等），禁止裸色值
- 暗色主题：确保所有组件在暗色主题下可读
- 响应式：使用 `sm:`/`md:`/`lg:` 断点，移动端优先
- 中文排版：标题使用 `font-bold tracking-wide`，正文使用 `leading-relaxed`

## Risks / Trade-offs

- **[风险] 视觉修改可能意外改变布局** → 每次修改后运行 dev server 目视检查
- **[风险] 大规模 CSS 类名修改可能引入不一致** → 先建立标准，按标准批量替换
- **[风险] 审查可能遗漏运行时状态（loading/error/empty）** → 审查时特别关注条件渲染分支
- **[权衡] 代码审查 vs 运行截图** → 代码审查为主，关键页面才运行验证，节省时间
