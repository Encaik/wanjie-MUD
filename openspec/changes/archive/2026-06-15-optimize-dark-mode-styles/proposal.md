## Why

暗色模式是游戏的重要外观选项，但当前暗色模式的呈现效果远差于亮色模式，主要有三个层面的问题：

**1. 视觉层次扁平化。** 当前暗色主题的 CSS 变量色度极低（0.02~0.04），各层之间明度差过小——背景（L=0.18）与卡片（L=0.22）仅差 4 个百分点，肉眼几乎无法区分。亮色模式下丰富的层次感（米色底、卡片白、棕色边框）在暗色下全部坍缩为一片深黑。

**2. 次级文本对比度不足。** `--muted-foreground` 在暗色下为 `oklch(0.65 0.03 60)`，与深色背景（L=0.18）的 WCAG 对比度仅约 4.5:1，对于 CJK 小字号文本严重不足。`--secondary-foreground`（L=0.88）同样缺乏与正文的区分度——正文（L=0.92）和次要（L=0.88）之间的明度差太小。

**3. 文本视觉过小。** 约 64 个文件中组件大量使用 `text-sm` 和 `text-xs`，暗色模式下小字号 + 低对比度的组合造成严重阅读困难。亮色模式的米色纸质感底色天然提供了舒适的阅读背景，暗色模式下深黑背景却没有相应增大字号或增强对比度来补偿。

**4. 硬编码色盘不一致。** 大量组件仍使用 `text-emerald-600 dark:text-emerald-400` 这类硬编码色盘 + `dark:` 回退模式，而非语义化 CSS 变量。不同组件的 dark 回退值亮度选择参差不齐（有的用 `-400`，有的用 `-300`），导致页面整体暗色观感不统一。

## What Changes

优化暗色模式下所有语义 CSS 变量的值，增强对比度和层次感，同时修复文本大小问题：

- **调整 35+ 个暗色 CSS 变量**：提升各层之间的明度差，增加色度营造温度感，确保 WCAG AA 对比度标准（至少 4.5:1 正文，3:1 大文本），同时保留修仙题材的"墨色古籍"调性
- **增强视觉层次**：背景 → 卡片 → 边框 三层明度梯度从 4% 差提升到 8-12% 差，卡片与弹出层有明显区分
- **提高次级文本可读性**：`muted-foreground` 明度从 0.65 提升至 0.70+，`secondary-foreground` 与 `foreground` 的明度差拉大
- **增大基础字号**：body 基准字号从默认 16px 确保至少 16px，小号文本（text-xs/text-sm）在暗色模式下的实际对比度通过变量调整后满足可读性
- **品质颜色暗色调优**：确保 8 级品质颜色在暗色背景下饱和度适中、不刺眼、不褪色
- **游戏领域色暗色调优**：6 个游戏语义色的暗色值微调，确保在深色背景上鲜艳但不刺眼

## Capabilities

### Modified Capabilities
- `style-tokens`: CSS 变量暗色组值更新（`themes.css` `.dark` 块）
- `ui-theme-compliance`: 暗色模式语义色一致性的新验证场景
- `visual-standards`: 新增暗色模式可读性标准要求

## Impact

- **`src/app/styles/themes.css`**: `.dark` 块中约 40+ 个 CSS 变量值重设，这是核心变更文件
- **`src/app/styles/base.css`**: body 基础字号确保
- **`src/app/styles/tokens.css`**: 无结构变更，仅需验证桥接仍然正确
- **`src/modules/theme/data/defaultTheme.ts`**: 如果 `DEFAULT_DARK_THEME` 常量的值与 CSS 中同步，需要一致更新
- **无破坏性变更**：不改变变量名，不改变 HTML 结构，不改变主题切换逻辑，不影响亮色模式
- **视觉影响范围**：所有使用语义 CSS 变量的组件（~64 个文件中的 `dark:` 变体）自动受益于变量值更新；硬编码色盘的组件行为不变
