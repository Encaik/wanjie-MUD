# game-design/ — 使用说明

## 作用

`game-design/` 是万界修行录项目的**游戏设计知识库**，记录游戏系统的全局设计视图。它是两位协作者共同参考的"设计合约"——通过阅读这里的文档，任何一方都能了解游戏设计的全貌和演变历史。

## 与 OpenSpec 的关系

| 维度 | game-design/ | openspec/changes/ |
|------|-------------|-------------------|
| **记录什么** | 系统最终设计（"系统长什么样"） | 变更过程（"改了什么、为什么改"） |
| **粒度** | 系统级（稳态设计决策） | 变更级（每次修改） |
| **维护时机** | 设计发生变动时同步更新 | 每个 Change 都会创建 |

两者**互为补充**，不互相替代。

## 同步规则

### 创建 OpenSpec Change 时
如果变更涉及游戏设计改动，同步更新 `game-design/` 对应章节。例如：
- 修改了属性系统 → 更新 `characters.md`
- 调整了战斗流程 → 更新 `combat.md`
- 改了世界生成逻辑 → 更新 `worlds.md`

### 归档 OpenSpec Change 时
追加一条记录到 `changelog.md`，格式如下：

```markdown
## [日期] - 变更标题

- **改动**: 设计变更内容描述
- **原因**: 为什么做这个变更
- **涉及章节**: 被修改的章节列表
- **开发者**: 执行变更的开发者
```

### 非 Change 小改动
顺手更新对应章节，不做强制要求。

## 每章内容模板

所有系统章节遵循统一的五区块模板：

1. **设计原则** — 为什么这么设计，关键设计决策和理由
2. **核心概念** — 系统的核心数据和流程，关键术语定义
3. **数据模型** — 核心类型定义（标注代码位置），关键关系
4. **模块映射** — 概念到代码位置的映射表
5. **相关文档** — 详细设计文档和 OpenSpec Change 链接
