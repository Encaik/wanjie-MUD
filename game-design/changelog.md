# 设计变更日志

## 2026-06-18 - 突破与心魔系统重设计（提案阶段）

- **改动**: 将突破和心魔两个独立系统融合为"心魔突破战"三阶段流程（心魔显形→心魔交锋→心魔炼化）；引入世界观V3 CoreStats属性对心魔/突破的影响映射；设计全屏沉浸式心魔战场UI；增加心魔动态生成引擎（世界观×心魔类型×玩家数据）；心魔图鉴长期记录和宿敌系统
- **原因**: 当前突破是纯概率判定零操作感，心魔只是5种固定事件的简单弹窗选择，两者完全割裂；突破缺乏仪式感和世界观差异化体现
- **涉及章节**: breakthrough-demon-redesign.md（新增）、characters.md（CoreStats映射）、worlds.md（世界观心魔差异）
- **开发者**: AI Agent (Claude)

## 2026-06-16 - 新手引导 + 统计系统 + 任务面板重构（设计中）

- **改动**: 设计统一统计追踪系统（`core/statistics/`）、事件驱动分阶段新手引导（5 阶段 9 步骤）、Tab 式任务中心面板；统计触发从分散式改为 EventBus 事件驱动；初始物品归入引导体系
- **原因**: 统计触发分散在 5+ 个 Hook 中各自为政；新手引导与业务逻辑耦合，缺乏教学弹窗和分阶段设计；任务面板简陋不支持多任务系统
- **涉及章节**: 统计系统、任务系统（design.md 待补充为正式章节）
- **OpenSpec**: `openspec/changes/tutorial-statistics-quest-redesign/`
- **开发者**: AI Agent (Claude)

## 2026-06-15 - 修炼功能修复与消息面板统一

- **改动**: 主角初始化时发放初始资源（200灵石 + 丹药）；右侧栏聊天/系统消息/公告合并为统一消息面板，新增消息类型筛选器；修复新手任务灵石奖励不创建新条目的 Bug
- **原因**: 路由驱动重构后背包初始化为空导致修炼功能不可用；三栏消息分散不符合传统 MUD 体验
- **涉及章节**: systems.md（修炼系统初始资源配置）, ui.md（右侧栏消息面板布局）
- **开发者**: AI Agent (Claude)
