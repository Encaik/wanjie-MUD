# 属性上限与词条持久性设计方案

> **状态**: ✅ 已迁移到 `game-design/characters.md`

## 1. 问题分析

### 1.1 当前问题

**问题描述**：玩家初始选择的角色（化身）在进入游戏后，其词条和属性优势会在几次修炼后被磨平，导致角色选择失去意义。

**数值分析**：

| 指标 | 当前值 | 说明 |
|------|--------|------|
| 基础属性 | 50 | 所有角色的初始属性 |
| 词条加成 | +5~+35 | 根据词条品质和组合，传说最高约+35 |
| 初始属性范围 | 50~85 | 天命之子可能达到85 |
| 等级1属性上限 | 100 | `100 + (1-1) × 6 = 100` |
| 100级属性上限 | 694 | `100 + (100-1) × 6 = 694` |
| 每次闭关属性增长 | 1~4点 | 10倍/100倍/1000倍闭关都增长 |
| 100级后上限增长 | 每级+6 | 持续增长 |

**问题根因**：
1. **属性上限过高**：当前等级1属性上限100，而初始属性最高才85，只比上限低15点
2. **修炼可突破上限**：随着等级提升，上限持续增长，修炼获得的属性可以无限增长
3. **词条加成比例过低**：传说词条最高+35，在100级时上限694，词条加成只占5%
4. **无属性类型分化**：所有属性都是平等可增长的，没有保留初始差异的机制

### 1.2 期望行为

玩家选择化身时应该感受到：
- **词条品质确实决定潜力**：传说词条的角色应该有更高的成长天花板
- **初始属性差异有意义**：幸运高的角色在探索中有优势，不会被几次修炼抹平
- **定位系统真正影响玩法**：战斗型角色在战斗中有明显优势

## 2. 设计方案

### 2.1 核心设计理念

**分层属性上限系统**：将属性分为「基础属性」和「成长属性」两部分。

- **基础属性**：由初始词条决定，上限较高但固定，不会随等级提升而增长
- **成长属性**：通过修炼获得，有等级限制，但可以通过其他方式突破

### 2.2 详细设计方案

#### 方案A：固定属性上限（推荐）

**核心思路**：为每个属性设置一个由词条决定的「属性上限」，修炼获得的属性不能超过这个上限。

**属性上限计算公式**：
```
属性上限 = 基础上限(100) + 词条加成上限 × 等级系数

其中：
- 基础上限 = 100（固定）
- 词条加成上限 = 该属性所有词条正向加成之和 × 1.5
- 等级系数 = 1 + (level - 1) × 0.1  （每级+10%，上限2.0）
```

**数值示例**：

| 角色类型 | 体质词条加成 | 体质属性上限(1级) | 体质属性上限(50级) | 体质属性上限(100级) |
|----------|--------------|-------------------|-------------------|---------------------|
| 普通词条 | +5 | 100 + 5×1.5 = 107 | 100 + 7.5×5.9 = 144 | 100 + 7.5×10.9 = 182 |
| 稀有词条 | +15 | 100 + 15×1.5 = 123 | 100 + 22.5×5.9 = 233 | 100 + 22.5×10.9 = 345 |
| 传说词条 | +35 | 100 + 35×1.5 = 153 | 100 + 52.5×5.9 = 410 | 100 + 52.5×10.9 = 672 |

**优点**：
- 词条品质决定属性上限的差异明显
- 计算简单，易于理解
- 保留了一定的成长空间

**缺点**：
- 高等级时词条差距仍然会被缩小
- 需要修改属性应用逻辑

#### 方案B：词条决定属性成长率

**核心思路**：词条不仅决定初始属性，还决定修炼时属性增长的效率。

**属性增长公式**：
```
修炼属性增长 = 基础增长 × (1 + 词条加成系数)

其中：
- 基础增长 = 1~4点（当前值）
- 词条加成系数 = 该属性词条正向加成 / 100
```

**数值示例**：

| 角色类型 | 体质词条加成 | 体质增长系数 | 实际体质增长(基础2点) |
|----------|--------------|--------------|----------------------|
| 普通词条 | +5 | 1.05 | 2.1点 |
| 稀有词条 | +15 | 1.15 | 2.3点 |
| 传说词条 | +35 | 1.35 | 2.7点 |

**优点**：
- 不需要修改属性上限逻辑
- 词条差异体现在成长速度上

**缺点**：
- 差异不够明显
- 仍然无法解决"属性趋同"问题

#### 方案C：属性类型锁定（推荐实施）

**核心思路**：将属性分为「固定属性」和「可成长属性」，修炼只能增加「可成长属性」。

**属性分配规则**：
```
- 固定属性 = 基础50 + 词条加成（不可通过修炼提升）
- 可成长属性 = 初始0，通过修炼提升，上限=等级×2
```

**数值示例**：

| 角色类型 | 词条加成范围 | 固定属性(1级) | 可成长上限(1级) | 总属性(1级) | 总属性(50级) | 总属性(100级) |
|----------|--------------|---------------|-----------------|-------------|--------------|---------------|
| 普通词条 | +8~+12 | 58~62 | 2 | 60~64 | 160~164 | 260~264 |
| 优秀词条 | +15~+20 | 65~70 | 2 | 67~72 | 167~172 | 267~272 |
| 稀有词条 | +25~+32 | 75~82 | 2 | 77~84 | 177~184 | 277~284 |
| 史诗词条 | +35~+45 | 85~95 | 2 | 87~97 | 187~197 | 287~297 |
| 传说词条 | +50~+65 | 100~115 | 2 | 102~117 | 202~217 | 302~317 |

**优势比例**：
| 对比 | 1级优势 | 50级优势 | 100级优势 | 100级绝对差距 |
|------|---------|----------|-----------|---------------|
| 传说 vs 普通 | 72% | 28% | 17% | 45点 |
| 史诗 vs 稀有 | 15% | 7% | 5% | 15点 |

**设计分析**：
- **固定属性差距永久保留**：传说角色的固定属性比普通角色高45点，这个差距不会随等级变化
- **可成长部分仍然存在**：虽然上限相同，但玩家需要努力才能填满
- **词条品质在选择时就有价值**：1级时73%的优势比例让选择非常有意义

**优点**：
- 完美保留词条带来的初始优势
- 词条差异在整个游戏过程中都可见
- 数值合理，传说优势明显但不过分

### 2.3 推荐方案：方案C（属性类型锁定）

经过权衡，我们选择**方案C**，原因：
1. **最大程度保留角色选择的意义**
2. **词条品质差异在游戏全程都可见**
3. **为定位系统提供数值支撑**：战斗型角色的体质上限更高

## 3. 详细设计

### 3.1 数据结构变更

**新增类型定义**：
```typescript
// 可成长属性结构
interface GrowthStats {
  体质: number;  // 可成长部分，当前值
  灵根: number;
  悟性: number;
  幸运: number;
  意志: number;
}

// 角色属性完整结构
interface CharacterStats {
  base: CharacterStatsBase;           // 固定属性（词条决定）
  growth: GrowthStats;               // 可成长属性（修炼获得）
}

// 获取最终属性
function getFinalStats(stats: CharacterStats): CharacterStatsBase {
  return {
    体质: stats.base.体质 + stats.growth.体质,
    灵根: stats.base.灵根 + stats.growth.灵根,
    悟性: stats.base.悟性 + stats.growth.悟性,
    幸运: stats.base.幸运 + stats.growth.幸运,
    意志: stats.base.意志 + stats.growth.意志,
  };
}
```

### 3.2 成长上限计算

```typescript
// 可成长属性上限计算
function getGrowthStatCap(level: number): number {
  // 每级增加2点可成长上限
  return level * 2;
}

// 固定属性上限计算（用于机缘等特殊加成）
function getFixedStatCap(baseStat: number, level: number): number {
  // 固定属性上限 = 基础值 × (1 + level × 0.03)
  return Math.floor(baseStat * (1 + level * 0.03));
}
```

### 3.3 属性应用逻辑变更

```typescript
// 修改后的 applyStatChanges
function applyStatChanges(
  currentStats: CharacterStats,
  changes: Partial<CharacterStats>,
  level: number,
  statCapBonuses: Partial<CharacterStats> = {}
): CharacterStats {
  const newGrowth = { ...currentStats.growth };
  const growthCap = getGrowthStatCap(level);
  
  for (const [key, value] of Object.entries(changes)) {
    if (key === 'growth' && typeof value === 'object') {
      // 处理成长属性变化
      for (const [stat, change] of Object.entries(value)) {
        if (change !== undefined) {
          const newValue = newGrowth[stat as keyof GrowthStats] + change;
          // 限制在可成长上限内
          newGrowth[stat as keyof GrowthStats] = Math.max(0, Math.min(newValue, growthCap));
        }
      }
    }
  }
  
  return {
    base: currentStats.base,
    growth: newGrowth,
  };
}
```

### 3.4 界面显示变更

**属性面板显示格式**：
```
体质: 85 + 30/200
    [========>       ] 30/200

解读：
- 85：固定属性（词条决定）
- 30：当前可成长属性
- 200：可成长上限（等级×2 = 100×2）
```

### 3.5 兼容旧存档

```typescript
// 存档迁移函数
function migrateOldStats(oldStats: CharacterStatsBase): CharacterStats {
  // 假设旧存档中所有属性都是固定属性
  // 固定值 = oldStats × 0.65（保留65%）
  // 可成长值 = oldStats × 0.35（35%变为可成长）
  
  const base: CharacterStatsBase = {
    体质: Math.floor(oldStats.体质 * 0.65),
    灵根: Math.floor(oldStats.灵根 * 0.65),
    悟性: Math.floor(oldStats.悟性 * 0.65),
    幸运: Math.floor(oldStats.幸运 * 0.65),
    意志: Math.floor(oldStats.意志 * 0.65),
  };
  
  const growth: GrowthStats = {
    体质: Math.floor(oldStats.体质 * 0.35),
    灵根: Math.floor(oldStats.灵根 * 0.35),
    悟性: Math.floor(oldStats.悟性 * 0.35),
    幸运: Math.floor(oldStats.幸运 * 0.35),
    意志: Math.floor(oldStats.意志 * 0.35),
  };
  
  return { base, growth };
}
```

## 4. 数值平衡调整

### 4.1 词条加成建议范围

为确保方案C的数值合理，建议调整词条加成范围：

| 品质 | 当前加成范围 | 建议加成范围 | 说明 |
|------|--------------|-------------|------|
| 普通 | +2~+4 | +8~+12 | 大幅提高 |
| 优秀 | +3~+5 | +15~+20 | 大幅提高 |
| 稀有 | +4~+7 | +25~+32 | 大幅提高 |
| 史诗 | +6~+10 | +35~+45 | 大幅提高 |
| 传说 | +8~+12 | +50~+65 | 大幅提高 |

### 4.2 修炼属性增长调整

当前闭关修炼直接增长属性，建议改为只增长经验/等级，属性增长通过突破事件触发：

| 事件 | 属性增长 | 触发条件 |
|------|----------|----------|
| 境界突破 | +3~+10点（随机分配到可成长属性） | 突破成功 |
| 天人交感 | +5~+15点（全属性） | 特殊闭关效果 |
| 大彻大悟 | +3~+8点（主属性） | 顿悟效果 |

**注意**：闭关修炼仍然会获得经验，但只有特定事件才会增加属性。

### 4.3 成长效率对比（最终方案）

| 角色类型 | 词条加成 | 1级总属性 | 50级总属性(理论) | 100级总属性(理论) |
|----------|----------|-----------|------------------|-------------------|
| 普通词条 | +10 | 60 | 160 | 260 |
| 优秀词条 | +17.5 | 67.5 | 167.5 | 267.5 |
| 稀有词条 | +28 | 78 | 178 | 278 |
| 史诗词条 | +40 | 90 | 190 | 290 |
| 传说词条 | +57.5 | 107.5 | 207.5 | 307.5 |

**优势比例**：
- 1级时：传说/普通 = 107.5/60 = 1.79倍（79%优势）
- 100级时：传说/普通 = 307.5/260 = 1.18倍（18%优势）
- 100级绝对差距：307.5 - 260 = 47.5点

**设计评价**：
- 传说词条优势在1级时非常明显（79%），让选择非常有意义
- 即使到100级，仍然保持18%优势和47点绝对差距
- 可成长属性上限200点（100级），给玩家努力目标

## 7. 附录

### A. 设计原理

**为什么词条应该持久影响角色？**
- 角色扮演游戏的核心乐趣之一是"选择有意义"
- 如果所有选择都可以被后续行为抹平，选择就失去了意义
- 修仙题材中，"根骨"、"悟性"等先天资质本就应该是难以改变的

**为什么选择方案C而不是其他方案？**
- 方案A/B仍然会随等级提升而缩小差距
- 方案C通过分层设计，永久保留词条的基础优势
- 虽然改动较大，但能实现最佳游戏体验
