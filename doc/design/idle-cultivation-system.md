## 1. 系统概述

### 1.1 设计目标

**核心目标**: 避免玩家因资源不足导致的卡关，同时保持在线玩法的吸引力。

**设计原则**:
1. **防卡关**: 离线期间自动积累基础资源，确保回归后可继续游戏
2. **收益适度**: 离线收益约为在线效率的 30-50%，鼓励在线游玩
3. **上限保护**: 最大累计 8 小时，防止无限挂机
4. **资源平衡**: 离线不能获得灵石（在线修炼需要消耗灵石）
5. **爬塔关联**: 爬塔层数决定挂机掉落来源

### 1.2 核心变更（v2.0）

**问题**: 原设计中修炼模式会产出灵石，但在线修炼需要消耗灵石，导致离线白嫖。

**解决方案**:
1. **修炼模式**: 仅获得经验，不获得灵石
2. **灵石来源**: 仅通过战斗、机缘、爬塔获得
3. **新增巡塔模式**: 基于爬塔层数获得挂机收益

### 2.2 状态定义

| 状态 | 进入条件 | 退出条件 | 行为 |
|------|----------|----------|------|
| IDLE | 初始状态/领取完成 | 玩家选择挂机模式 | 等待玩家操作 |
| SELECTING | 玩家点击挂机按钮 | 玩家确认模式 | 展示模式选项和预期收益 |
| CULTIVATE | 选择修炼模式 | 离线时间结束/上线 | 积累经验（无灵石） |
| RECOVER | 选择恢复模式 | 离线时间结束/上线 | 恢复HP/MP/体力 |
| PATROL | 选择巡塔模式 | 离线时间结束/上线 | 从掉落池获取物品 |
| COMPLETE | 上线时检测 | 玩家确认 | 展示挂机收益 |
| CLAIMING | 玩家点击领取 | 领取完成 | 发放奖励到背包 |

### 3.2 修炼模式收益计算（重构）

```typescript
/**
 * 计算修炼模式收益（v2.0 重构）
 * 
 * ⚠️ 核心变更：移除灵石收益
 * 
 * 设计原则：
 * 1. 仅获得经验，不获得灵石
 * 2. 避免离线白嫖灵石的问题
 * 3. 灵石需要通过战斗/机缘/爬塔获得
 */
function calculateCultivateRewards(
  playerLevel: number,
  worldType: WorldType,
  offlineDuration: number,
): IdleRewards {
  // 有效时长（上限保护）
  const effectiveDuration = Math.min(offlineDuration, CONFIG.maxOfflineDuration);
  
  // 计算小时数
  const hours = effectiveDuration / (60 * 60 * 1000);
  
  // 基础经验 = (基础值 + 等级系数) * 小时数 * 效率
  const baseExp = (CONFIG.cultivateBaseExp + playerLevel * CONFIG.cultivateExpPerLevel);
  const expGain = Math.floor(baseExp * hours * CONFIG.offlineEfficiency);
  
  // 世界类型加成
  const worldMultiplier = getWorldMultiplier(worldType);
  const finalExp = Math.floor(expGain * worldMultiplier);
  
  // ⚠️ 移除灵石收益 ❌
  // 原因：在线修炼需要消耗灵石，离线不能白嫖灵石
  const spiritStoneGain = 0;
  
  return {
    experience: finalExp,
    spiritStones: spiritStoneGain,  // 始终为 0
    hp: 0,
    mp: 0,
    stamina: 0,
    materials: [],
    fragments: [],
  };
}
```

### 3.3 巡塔模式收益计算（新增）

```typescript
/**
 * 计算巡塔模式收益（v2.0 新增）
 * 
 * 设计原则：
 * 1. 灵石收益基于爬塔层数
 * 2. 物品收益来自掉落池
 * 3. 无经验收益（鼓励在线修炼获取经验）
 */
function calculatePatrolRewards(
  maxFloor: number,
  dropPool: DropPool,
  offlineDuration: number,
): IdleRewards {
  const effectiveDuration = Math.min(offlineDuration, CONFIG.maxOfflineDuration);
  const hours = effectiveDuration / (60 * 60 * 1000);
  const intervals = Math.floor(effectiveDuration / CONFIG.patrolDropInterval);
  
  // 灵石收益（基于层数，从掉落池获取）
  const spiritStoneRate = CONFIG.patrolSpiritStoneBase + maxFloor * CONFIG.patrolSpiritStonePerFloor;
  const spiritStones = Math.min(
    dropPool.totalSpiritStones,
    Math.floor(spiritStoneRate * hours * CONFIG.patrolEfficiency)
  );
  
  // 物品收益（从掉落池随机获取）
  const fragments: DropPoolItem[] = [];
  const materials: DropPoolItem[] = [];
  
  const availableItems = dropPool.items.filter(
    item => item.sourceFloor <= maxFloor
  );
  
  const dropChance = Math.min(
    0.8,
    CONFIG.patrolDropChanceBase + maxFloor * CONFIG.patrolDropChancePerFloor
  );
  
  for (let i = 0; i < intervals && availableItems.length > 0; i++) {
    if (Math.random() < dropChance) {
      const index = Math.floor(Math.random() * availableItems.length);
      const item = availableItems.splice(index, 1)[0];
      
      if (item.type === 'fragment') {
        fragments.push(item);
      } else {
        materials.push(item);
      }
    }
  }
  
  return {
    experience: 0,  // 巡塔模式无经验
    spiritStones,
    hp: 0,
    mp: 0,
    stamina: 0,
    materials,
    fragments,
  };
}
```

## 4. 边界条件处理

### 4.1 时间边界

| 边界条件 | 处理方式 |
|----------|----------|
| 离线时长 < 5分钟 | 不计算收益，显示"在线时间过短" |
| 离线时长 > 8小时 | 按8小时计算，显示"已达到最大累计时长" |
| 离线时长为负数 | 视为0，记录错误日志 |
| lastLogoutTime 为 0 | 视为首次登录，不计算收益 |

### 4.2 资源边界

| 边界条件 | 处理方式 |
|----------|----------|
| 经验值溢出 | 正常处理（已有 overflowExperience 机制） |
| 巡塔时掉落池为空 | 无物品收益，仅获得基础灵石 |
| 巡塔时爬塔层数为0 | 无收益，提示"先去爬塔积累掉落" |
| HP/MP 已满 | 不恢复，不浪费计算 |
| 体力已满 | 不恢复 |

### 4.3 模式边界

| 边界条件 | 处理方式 |
|----------|----------|
| 修炼模式但无灵石 | 正常进行（修炼模式不消耗灵石） |
| 巡塔模式但未爬塔 | 允许选择，但提示收益会很低 |
| 恢复模式但状态已满 | 允许选择，但提示收益会很低 |
