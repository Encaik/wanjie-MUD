# 机缘系统测试文档

## 系统概述

机缘系统是游戏的核心玩法之一，玩家进入秘境地图探索格子，遭遇敌人、获取奖励，最终挑战Boss完成探索。

## 核心环节

### 1. 地图生成系统 (Grid Generation)
- **入口**: `generateAdventureGrid(config, worldType)`
- **功能**:
  - 根据配置生成指定大小的地图
  - 按比例分配各种格子类型
  - 确保关键格子类型的最小数量
  - 设置起点（第一行中间）和Boss点（最后一行中间）
  - 大地图（81格以上）生成传送门

### 2. 格子类型系统 (Cell Types)
| 类型 | 比例 | 最小数量 | 功能 |
|------|------|----------|------|
| treasure | 10% | 1 | 宝箱，获取资源和物品 |
| enemy | 18%+ | 2 | 普通敌人 |
| elite | 3% | 0 | 精英敌人，+2等级 |
| miniboss | 1% | 1 | 小Boss，+5等级 |
| boss | 固定1个 | 1 | 最终Boss，+8等级 |
| event | 15% | 1 | 随机事件 |
| rest | 12% | 1 | 休息恢复HP/MP |
| empty | 41%- | - | 空格，可通过 |
| portal | 大地图 | - | 传送门，成对出现 |

### 3. 移动系统 (Movement)
- **规则**: 只能移动到相邻格子（上下左右）
- **起点**: 第一行中间位置，自动标记为已访问
- **传送门**: 点击后传送到配对的传送门位置
- **可重复访问**: 空格、已清除的格子可重复走过

### 4. 战斗系统 (Battle)
- **战斗流程**: 
  1. 计算双方属性（攻击、防御、HP、MP）
  2. 回合制交替攻击
  3. 触发功法技能（25%概率）
  4. 暴击/闪避判定
  5. 伤害上限限制（防止秒杀）
- **胜负判定**: HP归零方失败，超时按剩余HP判定

### 5. 奖励系统 (Rewards)
- **战斗奖励**:
  - 灵石：根据敌人等级和类型计算
  - 经验：根据敌人等级和类型计算
  - 物品：概率掉落丹药、功法、装备
  - 属性：概率获得属性提升
- **格子奖励**:
  - 宝箱：灵石 + 可能的额外物品
  - 事件：经验值
  - 休息：恢复HP/MP

### 6. 战利品系统 (Loot)
- **收集规则**: 
  - 战斗胜利获得的物品进入战利品背包
  - 未击败Boss时退出，可获得全部战利品
  - 被击败逃跑，丢失80%战利品
- **显示规则**:
  - 普通/稀有：直接显示名称
  - 史诗：显示"未知珍宝"
  - 传说：显示"稀世秘宝"

### 7. 难度选择系统 (Difficulty)
- **难度等级**: easy, normal, hard, nightmare
- **影响因素**:
  - 地图大小
  - 敌人等级范围
  - 奖励倍率
  - 传送门数量

### 8. 扫荡系统 (Quick Sweep)
- **条件**: 已通关该难度的秘境
- **奖励**: 直接获得奖励，跳过探索过程
- **限制**: 有每日扫荡次数限制

## 测试覆盖要求

### 必测场景
1. **地图生成**：各种配置下的地图生成正确性
2. **格子比例**：确保各种格子类型的合理分布
3. **传送门**：大地图传送门成对且可传送
4. **战斗平衡**：玩家与敌人的战斗结果合理
5. **奖励计算**：各种情况下的奖励正确
6. **战利品**：收集、显示、丢失逻辑正确
7. **移动规则**：相邻移动、传送门传送正确
8. **术语系统**：不同世界观的术语正确使用

### 边界条件
- 最小地图（5x5）
- 最大地图（30x30）
- 传送门阈值（9x9=81格）
- 战斗超时（100回合）
- 伤害上限（60%HP）

### 异常处理
- 无效移动
- 战斗失败处理
- 物品为空的显示
- 传送门数量不足
